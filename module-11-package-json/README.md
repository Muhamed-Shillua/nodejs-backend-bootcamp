# Module 11 – package.json & NPM

## Module Introduction

Every professional Node.js project relies on npm (Node Package Manager) and the `package.json` file.

**Why this module matters:**

* `package.json` defines project metadata
* Manages dependencies (external libraries)
* Defines scripts for automation
* Keeps track of versions and engines
* Facilitates collaboration and deployment

In real backend projects, npm and `package.json` are essential for:

* Installing frameworks like Express
* Managing dev tools (e.g., ESLint, Nodemon)
* Building automated workflows
* Handling version control of dependencies

By mastering this module, you will be able to manage professional Node.js projects efficiently.

## Lesson 01 – Creating package.json

Initialize a new Node.js project:

```bash
npm init
# or auto-fill defaults
npm init -y
```

Example `package.json`:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "My first Node.js app",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  },
  "author": "Muhamed Shillua",
  "license": "MIT",
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^2.0.22"
  }
}
```

## Lesson 02 – Dependencies vs DevDependencies

* **dependencies** → required to run the app (e.g., Express, Axios)
* **devDependencies** → required only during development (e.g., ESLint, Nodemon)

Install packages:

```bash
npm install express           # dependency
npm install --save-dev nodemon # dev dependency
```

## Lesson 03 – Scripts

Scripts automate tasks.

Example:

```json
"scripts": {
  "start": "node index.js",
  "dev": "nodemon index.js",
  "test": "mocha"
}
```

Run scripts:

```bash
npm run dev
```

## Lesson 04 – Versioning

`version`: "1.0.0" → semantic versioning (semver)

Format: `MAJOR.MINOR.PATCH`

* PATCH → bug fixes
* MINOR → new features (backward-compatible)
* MAJOR → breaking changes

Dependencies also follow semver with `^` or `~`:

```json
"express": "^4.18.2"  // compatible with 4.x.x
```

## Lesson 05 – npm Commands

| Command                 | Purpose                        |
| ----------------------- | ------------------------------ |
| npm install <package>   | Install dependency             |
| npm uninstall <package> | Remove package                 |
| npm update              | Update packages                |
| npm audit               | Check security vulnerabilities |
| npm init                | Create package.json            |
| npm list                | Show installed packages        |
| npm run <script>        | Run script from package.json   |

## Lesson 06 – Managing Node Versions

Use `engines` in `package.json`:

```json
"engines": {
  "node": ">=18.0.0"
}
```

Ensures all developers use compatible Node.js versions.

## Lesson 07 – package-lock.json

* Auto-generated by npm
* Locks exact versions of dependencies
* Ensures consistent installs across environments

`npm install` reads `package-lock.json` to install exact versions.

## Lesson 08 – Real Backend Example – Project Setup

Project structure:

```
my-app/
├─ index.js
├─ package.json
├─ package-lock.json
├─ node_modules/
├─ src/
│  ├─ routes/
│  └─ controllers/
└─ .gitignore
```

Setup workflow:

```bash
npm init -y
npm install express
npm install --save-dev nodemon
```

Run server:

```bash
npm run dev
```

## Common Developer Mistakes

1️⃣ Forgetting to add dependencies → app breaks in production
2️⃣ Using global installs instead of local project installs
3️⃣ Not committing package-lock.json → version mismatches
4️⃣ Overwriting scripts accidentally
